Princípio 2: Operável

Os componentes da interface e a navegação devem ser operáveis por todos os usuários, independentemente de sua capacidade física ou da tecnologia utilizada.

"Componentes de interface de usuário e navegação devem ser operáveis."
— WCAG 2.2

Diretrizes Principais

2.1 Acessível por Teclado

Torne toda a funcionalidade disponível a partir de um teclado, sem exigir tempos específicos para pressionamentos de teclas.

Demonstração: Navegação por Teclado

Armadilha de Teclado

Exemplo ruim: Armadilha de teclado

Uma armadilha de teclado ocorre quando o usuário pode entrar em uma área da página usando o teclado, mas não consegue sair dela usando apenas o teclado.

Experimente: Entre nesta "armadilha" usando a tecla Tab e tente sair dela (você não conseguirá sem usar o mouse).

Os usuários que navegam apenas por teclado ficariam presos nesta seção.

Exemplo bom: Navegação por teclado adequada

Todos os controles interativos devem ser acessíveis via teclado, e o usuário deve poder navegar para dentro e para fora de todas as áreas da página.

Experimente: Esta seção permite navegação normal com Tab.

Usuários podem navegar por todos os elementos usando o teclado sem ficar presos.

/* JavaScript que cria uma armadilha de teclado (prática ruim) */ element.addEventListener('keydown', function(e) { if (e.key === 'Tab') { e.preventDefault(); // Impede a navegação normal com Tab // Código para mover o foco entre elementos predefinidos... } }); /* Em vez disso, permita que o navegador gerencie o foco (boa prática) */ // Não manipule eventos de Tab, exceto em casos específicos como widgets de diálogo modal, // onde você deve fornecer uma maneira óbvia de sair (ex: botão "Fechar" ou tecla ESC)
