Princípio 3: Compreensível

A informação e a operação da interface do usuário devem ser compreensíveis para todos. Isso significa que o conteúdo deve ser claro e a funcionalidade previsível.

"A informação e a operação da interface de usuário devem ser compreensíveis."
— WCAG 2.2

Diretrizes Principais

3.1 Legível

Torne o conteúdo do texto legível e compreensível para todos os usuários.

Demonstração: Linguagem Clara

Exemplo ruim: Linguagem complexa e técnica

A implementação hipotética de paradigmas de acessibilidade digital mediante metodologias de semantização estrutural em HTML5 possibilita a obtenção de resultados otimizados em termos de interatividade assistiva para dispositivos de tecnologia adaptativa.

O exemplo acima usa linguagem desnecessariamente complexa e jargão técnico, tornando o conteúdo difícil de entender para muitos usuários.

Exemplo bom: Linguagem clara e direta

Usar HTML5 com estrutura semântica clara torna seu site mais acessível. Isso ajuda pessoas que usam leitores de tela e outras tecnologias assistivas a entenderem melhor seu conteúdo.

O exemplo acima transmite a mesma informação, mas de forma clara e direta, facilitando a compreensão para todos os usuários.

Boas práticas para conteúdo legível:

  • Use linguagem clara e direta, evitando jargões desnecessários
  • Organize o conteúdo com cabeçalhos e listas para facilitar a leitura
  • Defina o idioma principal do documento e de seções em outros idiomas
  • Explique abreviações e termos técnicos na primeira ocorrência
  • Ofereça definições para palavras incomuns ou uso específico de palavras

3.2 Previsível

Faça com que as páginas da web apareçam e funcionem de maneira previsível para todos os usuários.

Demonstração: Operação Previsível

Exemplo ruim: Mudanças automáticas inesperadas

No exemplo acima, mudar a seleção causaria uma mudança de contexto automática (como um redirecionamento) sem avisar o usuário. Isso é desorientador e frustrante.

Exemplo bom: Controle do usuário sobre mudanças

No exemplo acima, a mudança de seleção não causa alteração automática. O usuário tem controle para confirmar sua escolha antes de qualquer mudança de contexto.

Boas práticas para interfaces previsíveis:

  • Mantenha navegação e elementos de interface consistentes em todas as páginas
  • Não inicie mudanças automáticas quando um elemento recebe foco
  • Não altere o contexto automaticamente quando um usuário insere dados
  • Forneça mecanismos para interromper ou controlar conteúdo em movimento
  • Avise usuários antes de abrir novas janelas ou guias

3.3 Assistência de Entrada

Ajude os usuários a evitar e corrigir erros em formulários e outras interações.

Demonstração: Mensagens de Erro Acessíveis

Exemplo ruim: Mensagens de erro inadequadas

Problemas neste exemplo:

  • Não indica quais campos são obrigatórios antes do envio
  • Mensagem de erro não é associada ao campo por tecnologias assistivas
  • Não há instruções claras sobre como corrigir o erro
  • O foco não retorna ao campo com erro

Exemplo bom: Mensagens de erro acessíveis

Este campo é obrigatório

Melhorias neste exemplo:

  • Indica claramente quais campos são obrigatórios antes do envio
  • Associa mensagens de erro ao campo usando aria-describedby
  • Fornece instruções claras sobre como corrigir o erro
  • Retorna o foco ao campo com erro

Checklist de Verificação

Verificações Importantes

Use esta lista para verificar se seu conteúdo atende ao princípio "Compreensível":

Critério Como Verificar
Idioma da página Verifique se o atributo lang está presente no elemento html
Navegação consistente Verifique se os menus de navegação estão na mesma ordem em todas as páginas
Identificação consistente Elementos com a mesma função devem ser identificados consistentemente
Ajuda contextual Forneça instruções para ajudar usuários a entenderem formulários complexos
Prevenção de erros Implemente verificações, confirmações e opção de reversão em ações importantes
Rótulos e instruções Todos os campos de formulário devem ter rótulos descritivos
