Princípio 1: Perceptível

As informações e os componentes da interface do usuário devem ser apresentados de maneira que possam ser percebidos por todos, independentemente de suas capacidades sensoriais.

"Informação e componentes da interface de usuário devem ser apresentados aos usuários em formas que eles possam perceber."
— WCAG 2.2

Diretrizes Principais

1.1 Alternativas em Texto

Forneça alternativas em texto para qualquer conteúdo não textual.

Demonstração: Texto Alternativo

Exemplo ruim: Imagem sem texto alternativo

Quando uma imagem não possui texto alternativo, leitores de tela podem anunciar apenas o nome do arquivo ou ignorar completamente a imagem.

Exemplo bom: Imagem com texto alternativo adequado

Com texto alternativo adequado, os usuários de leitores de tela recebem a mesma informação transmitida pela imagem.

<!-- Exemplo ruim: Sem texto alternativo --> <img src="grafico.jpg"> <!-- Exemplo bom: Com texto alternativo adequado --> <img src="grafico.jpg" alt="Gráfico mostrando estatísticas de acessibilidade web no Brasil em 2023">

1.2 Mídias com Base no Tempo

Forneça alternativas para mídias com base no tempo, como legendas, audiodescrição ou transcrição.

Mídia Acessível

Exemplo ruim: Vídeo sem legendas ou transcrição

Um vídeo sem legendas ou transcrição exclui pessoas surdas ou com deficiência auditiva do acesso ao conteúdo.

Exemplo bom: Vídeo com legendas e transcrição

Adicione legendas a vídeos e forneça transcrições para conteúdo em áudio e vídeo.

Exemplo de implementação com HTML5:

<video controls> <source src="video.mp4" type="video/mp4"> <track kind="captions" src="legendas.vtt" srclang="pt-BR" label="Português"> <track kind="descriptions" src="audiodescricao.vtt" srclang="pt-BR" label="Audiodescrição"> <p>Seu navegador não suporta vídeos HTML5. Aqui está <a href="video-transcript.html">uma transcrição do vídeo</a>.</p> </video> <details> <summary>Transcrição do vídeo</summary> <div> <p>[Pessoa falando]: "A acessibilidade digital é um direito de todos..."</p> <p>[Descrição da cena]: Demonstração de uma pessoa usando leitor de tela</p> <!-- Transcrição completa --> </div> </details>

1.3 Adaptável

Crie conteúdo que possa ser apresentado de diferentes maneiras sem perder informação ou estrutura.

Demonstração: Estrutura Semântica

Exemplo ruim: Uso excessivo de divs sem semântica

Rodapé

Estrutura semântica clara permite que tecnologias assistivas identifiquem corretamente cada seção.

<!-- Exemplo ruim: Sem semântica --> <div> <div>Meu Site</div> <div> <div>Navegação</div> <div>Link 1</div> <div>Link 2</div> </div> <div> <div>Título</div> <div>Texto...</div> </div> </div> <!-- Exemplo bom: Com semântica HTML5 --> <header> <h1>Meu Site</h1> </header> <nav> <h2>Navegação</h2> <ul> <li><a href="#">Link 1</a></li> <li><a href="#">Link 2</a></li> </ul> </nav> <main> <h2>Título</h2> <p>Texto...</p> </main>

1.4 Discernível

Torne mais fácil para os usuários ver e ouvir o conteúdo, incluindo a separação do primeiro plano do plano de fundo.

Demonstração: Contraste de Cores

Exemplo ruim: Baixo contraste

Este texto tem um contraste insuficiente com o fundo, tornando-o difícil de ler para muitas pessoas, especialmente aquelas com baixa visão.

Relação de contraste: 1.5:1 (abaixo do mínimo recomendado de 4.5:1)

Exemplo bom: Alto contraste

Este texto tem contraste adequado com o fundo, tornando-o legível para a maioria das pessoas, incluindo aquelas com baixa visão.

Relação de contraste: 21:1 (bem acima do mínimo recomendado)

/* CSS com contraste insuficiente */ .baixo-contraste { background-color: #f8f8f8; color: #c0c0c0; /* Contraste insuficiente */ } /* CSS com contraste adequado */ .alto-contraste { background-color: #f8f8f8; color: #000000; /* Contraste adequado */ }

Outros aspectos do princípio "Discernível":

  • Tamanho do texto: O texto deve ser redimensionável até 200% sem perda de conteúdo ou funcionalidade
  • Imagens de texto: Evite usar imagens de texto quando o mesmo efeito visual pode ser obtido com texto estilizado
  • Controle de áudio: Forneça controles para pausar, parar ou ajustar o volume de qualquer áudio que reproduza automaticamente
  • Espaçamento do texto: Permita que usuários ajustem o espaçamento do texto sem perda de conteúdo ou funcionalidade

Checklist de Verificação

Verificações Importantes

Use esta lista para verificar se seu conteúdo atende ao princípio "Perceptível":

Critério Como Verificar
Todas as imagens têm texto alternativo Verifique se todas as tags <img> possuem atributo alt adequado
Contraste de cores adequado Use ferramentas como WAVE ou Colour Contrast Analyzer para verificar relações de contraste
Estrutura semântica correta Verifique se o HTML usa elementos semânticos (<header>, <nav>, <main>)
Conteúdo responsivo Teste em diferentes tamanhos de tela e níveis de zoom
Multimídia com alternativas Verifique se vídeos têm legendas e áudios têm transcrições
Informações não dependem apenas de cor Confirme que todas as informações transmitidas com cor também são disponíveis de outras formas
